# M61 Web 配置器重构规格

[English](M61_WEB_REFACTOR_SPEC.md)

## 1. 目标

把面向 Pico2W 的配置器改造成正式 M61 配置器，同时不能削弱已经实测的固件
实时发布配置。网页与固件必须共用一个有版本、可测试的管理协议。

## 2. 协议

首版保留 Feature Report ID，以减少浏览器传输层改动，但更换协议身份：

| Report | 方向 | M61 含义 |
| --- | --- | --- |
| `0xF6` | 主机到设备 | 命令封装：应用、保存、USB 重连，后续扩展维护命令 |
| `0xF7` | 设备到主机 | 当前 M61 配置 |
| `0xF8` | 设备到主机 | 固件与构建信息 |
| `0xF9` | 设备到主机 | 紧凑实时遥测 |

要求：

- 增加 M61 协议 magic 和 schema 版本。遇到 Pico 配置 v5 必须明确提示设备不兼容。
- WebHID payload 保持 63 字节，未使用字节全部为零。
- 多字节整数使用小端，布尔值只能是 `0` 或 `1`。
- 每条命令必须完整校验后才能改变任何状态。
- `0xF6`–`0xF9` 在本机处理，不能进入手柄 Feature 请求队列；其他 Report ID
  保持现有代理行为。
- `0xF9` 保留 RSSI/活动兼容前缀：第 1 字节为有符号 RSSI，第 2 字节 bit 7
  表示有效、bit 1 表示扬声器活动、bit 0 表示麦克风活动；后续为有版本的 M61 遥测。

## 3. 配置模型 v1

首版只包含已经有明确、可测试 M61 语义的设置：

| 字段 | 类型/范围 | 应用方式 | 持久化 |
| --- | --- | --- | --- |
| 启用麦克风 | bool | 立即；关闭时 USB 返回静音数据 | 显式保存 |
| 启用扬声器 | bool | 立即；不能关闭 HD 震动 | 显式保存 |
| 扬声器路由 | auto/mono/stereo | 立即 | 显式保存 |
| 自动重连 | bool | 立即作用于连接状态机 | 显式保存 |
| 启用状态灯 | bool | 立即；启用时回到自动状态 | 显式保存 |
| 触觉增益 | 校验过的定点范围 | 立即、饱和缩放 | 显式保存 |
| CPU governor | manual/realtime | 立即 | 显式保存 |
| CPU 档位/频率 | 安全档位或 320–400 MHz | 立即 | 显式保存 |

普通网页不能写入或保存 400 MHz 以上实验频率。会改变 USB 描述符的设置不进入 v1。

固件保存单一记录：magic、schema 版本、payload 长度、payload 和 CRC。无效或未知
记录回退到发布默认值，并通过遥测报告原因。只有显式保存才写 Flash，以控制磨损。

## 4. 遥测模型 v1

导出稳定产品数据，不能直接暴露内部结构体：

- 蓝牙状态、已保存手柄、配对/扫描状态和 RSSI 有效性；
- USB 已配置/挂起状态；
- 扬声器/麦克风活动、耳机状态、实际 mono/stereo 路由；
- 当前/请求 CPU 频率与 governor；
- 紧凑的队列深度/峰值/丢弃与编解码错误计数；
- 配置已加载/未保存/有效状态和最近管理错误。

计数器允许饱和。报告必须能在 HID 控制路径中无阻塞、无动态分配地生成。

## 5. 固件工作包

### F0 — 协议基础

- 新增 `m61_web_config.[ch]`，包含默认值、校验、运行时应用、持久化、报告编码
  和可做主机单测的纯函数。
- 在 HID GET/SET 回调中截获 `0xF6`–`0xF9`。
- USB 重连等破坏性操作延迟到任务上下文执行。
- 增加编解码、错误长度/版本/范围、本地报告截获、普通代理保持、CRC 保存/加载
  失败等主机测试。

### F1 — 接入现有 M61 设置

- 让麦克风、扬声器路由、自动重连、状态灯和 DVFS 由统一配置模块管理。
- 增加扬声器启用和运行时触觉增益，但不能改变队列大小、任务优先级、Codec 参数
  或内存放置。
- 没有有效持久化记录时保持当前发布默认值。

### F2 — 遥测与手柄管理

- 实现遥测 v1；硬件/API允许时增加连接状态 RSSI 采样。
- 增加异步配对/扫描/断开/忘记命令和状态轮询。
- USB 控制回调绝不能等待蓝牙扫描完成。

## 6. Web 工作包

### W0 — 协议和测试

- 分离传输、schema 和展示类型。
- 解码配置前先识别 M61 magic/版本。
- 为所有报告和错误响应增加 fixture 测试。
- 保留串行化/合并连续修改的即时应用和显式保存语义。

### W1 — M61 信息架构

- 替换产品/Pico 文案并更新 README/PWA 元数据。
- 把 Pico 字段列表改成手柄、音频、性能、设备、诊断几个区块或标签页。
- 同时显示配置值和实际生效值，并明确提示是否需要 USB 重连。
- 所有字符串保持中英文同步。

### W2 — M61 操作

- 增加配对、断开、忘记、健康遥测、诊断导出和带确认的 ISP 重启。
- 根据协议 capability bits 隐藏不可用功能，不能展示实际无效的控件。

## 7. 性能与安全门禁

固件验收要求：

- 现有离线测试和 Windows 完整发布构建通过；
- 发布编译器、Opus、WRAM 设置和构建清单不变；
- 音频逐帧路径不能新增分配、日志、Flash 访问或阻塞；
- RAM/ITCM 门禁继续通过；
- 实机满载音频不能新增编解码错误或队列丢弃；
- 配置应用/保存/重连模糊测试和掉电行为经过验证。

Web 验收要求 TypeScript 构建、协议 fixture 测试、Chromium 手工测试、断线重连恢复，
以及中英文字符串覆盖一致。

## 8. 交付顺序

1. F0 + W0：协议协商以及只读身份/配置往返。
2. F1 + W1：可用的 M61 配置、应用、保存和重连。
3. F2 + W2：配对、健康遥测、诊断和维护操作。
4. 实机验证并公开发布。
5. 逐项评估暂缓的 Pico 功能；没有固件语义和性能证据之前不能进入 UI。
