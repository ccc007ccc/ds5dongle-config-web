# M61 Web 配置器规格

[English](M61_WEB_REFACTOR_SPEC.md)

## 产品边界

本仓库是 M61 BL616/BL618 DualSense 接收器的配置与诊断应用。所有设置、文案、
资源、协议类型和文档都必须对应 M61 能力。React、WebHID 和 PWA 只是通用实现
技术，不形成第二套产品模式。

## 管理协议

| Report | 方向 | 含义 |
| --- | --- | --- |
| `0xF6` | 主机到 M61 | 应用、保存、USB 重连和维护命令 |
| `0xF7` | M61 到主机 | 当前有版本配置 |
| `0xF8` | M61 到主机 | 固件与可复现构建信息 |
| `0xF9` | M61 到主机 | 实时遥测和能力状态 |

所有报告使用 63 字节 payload。配置以 `M61C`、schema 版本、payload 长度和能力
位开始，多字节值使用小端。magic、版本、长度、命令或范围无效时，必须在改变
状态前拒绝。`0xF6` 到 `0xF9` 由 M61 本地处理，其他 DualSense Feature Report
继续代理给真实手柄。

## 配置 v1

schema 包含麦克风启用、扬声器启用、扬声器路由、自动重连、M61 状态灯、Q8
触觉增益、CPU governor、CPU 档位和安全自定义频率。只有 capability bit 存在时
网页才显示对应控件。普通网页不能写入超过实机验证上限 400 MHz 的频率。

修改立即、串行化应用；保存是独立操作，把一个带版本和 CRC 的记录写入 Flash。
记录无效时回退到实测发布默认值。Flash 和 USB 重连只能在任务上下文执行，不能
进入 HID 回调或实时音频路径。

## 遥测 v1

遥测导出稳定产品数据：蓝牙连接和 RSSI 有效性、USB 配置/挂起、扬声器与麦克风
活动、耳机和实际声道路由、当前/请求 CPU 频率、受限队列/丢弃/错误计数、配置
状态及最近管理错误。编码必须静态、无分配、无阻塞，并能安全运行在 USB 控制路径。

## 交付包

1. 协议基础：跨语言 fixture、本地报告截获、固件身份、配置回读、DVFS 应用和实机测试。
2. 统一配置：CRC 持久化并接入麦克风、扬声器、路由、重连、状态灯、触觉增益和 DVFS。
3. M61 界面：按 capability 展示手柄、音频、性能、设备、诊断区块，中英文完整对应。
4. 操作：配对、断开、忘记、诊断导出和带确认的 ISP 重启。
5. 硬件发布：满载音频、RAM/ITCM、重连、掉电恢复和 Chromium WebHID 验收。

## 性能与安全门禁

- 保持锁定 SDK、工具链、Opus 补丁栈、160 KiB WRAM、编译参数和构建清单。
- 音频逐帧路径不能新增分配、日志、Flash 访问或阻塞。
- 现有离线、协议、调度器、内存和完整固件构建全部通过。
- 实机满载音频不能新增 Codec 错误或队列丢弃。
- Web 测试、TypeScript、PWA、断线恢复和中英文 key 对等检查通过。
- 源码、UI、资源、元数据、README 和产品文档只包含 M61 产品概念；依赖锁文件
  不参与该检查。
