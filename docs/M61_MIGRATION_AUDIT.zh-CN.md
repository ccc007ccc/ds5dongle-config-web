# M61 迁移审计

[English](M61_MIGRATION_AUDIT.md)

## 结论

当前网页还不能配置 M61 固件。虽然设备描述符声明了 Feature Report
`0xF6` 到 `0xF9`，网页把它们定义成 Pico2W 私有管理协议，而 M61 固件
目前会把所有 Feature Report 转发给真实 DualSense。因此网页读不到配置
版本 5，写入甚至可能被发给手柄，而不是修改接收器设置。

逐项权威清单见
[`M61_FEATURE_GAP_MATRIX.csv`](M61_FEATURE_GAP_MATRIX.csv)。

## 现有 Web 功能

React/Vite PWA 的基础框架值得保留：

- WebHID 选设备和重新打开已授权设备；
- 串行化、合并连续修改的即时应用；
- 显式保存到 Flash、USB 重连、恢复默认、重新读取、校验和断线恢复；
- 固件版本、RSSI、扬声器/麦克风活动显示；
- 中英文、响应式界面、亮/暗/跟随系统主题、错误提示和 PWA 更新提示。

Pico 配置 v5 有 15 项：触觉增益、扬声器/耳机音量、扬声器增益、无操作
超时、关闭 Pico LED、轮询模式、音频缓冲长度、手柄身份、USB 序列号、
PS 快捷键、关闭麦克风、关闭扬声器、USB 唤醒和扳机削减。

## 现有 M61 固件功能

M61 已经拥有不少网页没有覆盖的能力：

- Classic Bluetooth 扫描、配对、已保存地址重连、安全、SDP、HIDP 双通道、
  断开和忘记手柄；
- 完整 DualSense USB HID 输入/输出与 Feature GET/SET 代理；
- 四声道 USB Audio OUT、HD 震动、扬声器/耳机自动路由、Opus 编码、麦克风
  Opus 解码、USB 音量/静音状态和音频健康数据；
- 运行时麦克风开关与 auto/mono/stereo 扬声器路由；
- manual/realtime DVFS、安全档位、320–400 MHz 自定义锁频、短时 boost、
  显式持久化和实验频率保护；
- USB 重初始化/重连、进入 ISP、RGB 状态灯、日志等级和大量蓝牙/USB/音频/
  性能诊断。

## 迁移原则

1. 保留 WebHID/PWA 架构以及可靠的应用、保存、重连状态机。
2. M61 管理报告必须先在接收器本地截获，再进入普通 DualSense Feature 代理。
3. 新增 M61 协议协商，绝不能把 Pico 配置 v5 当成 M61 配置。
4. 保留已经实测的发布性能路径。动态队列或缓冲配置只有通过基准后才能开放。
5. 使用正向 M61 概念，例如“启用麦克风”“启用状态灯”，不延续 Pico 的反向字段。
6. 会改变 USB 描述符的设置单独处理，并明确要求 USB 重连。
7. Flash 写入必须显式触发，完整记录校验通过后才能应用或保存。

## 首版边界

首个可用版本包括：协议协商、读取/应用/保存配置、USB 重连、固件信息、音频
活动、麦克风开关、扬声器路由、自动重连和安全 DVFS 设置。随后补蓝牙配对和
更丰富的健康遥测。Edge 身份、远程唤醒、扳机变换、快捷键和动态缓冲继续由
独立硬件与性能验证把关。
